Incidencias en Conciertos

Introdución

En el mundo de los eventos en vivo, los conciertos destacan como experiencias únicas que donde se reúnen multitudes para disfrutar de música y entretenimiento. Sin embargo, la organización de un concierto no está exenta de complicaciones y desafíos logísticos que pueden dar lugar a diversas incidencias, desde problemas técnicos en el escenario hasta dificultades en la gestión de la entrada y el público. Durante este trabajo se estará resolviendo el problema de las incidencias en conciertos a través de una solución basada en tecnologías web.

Este proyecto se fundamenta en el uso de herramientas y tecnologías como HTML, CSS, Javascript con Bootstrap, para abordar de manera efectiva los problemas y desafíos que suelen surgir en la producción y ejecución de conciertos en vivo.

Requerimientos del Cliente - Aplicación de Gestión de Incidencias en Conciertos

1. Página de Inicio:

Descripción: La página de inicio debe ser atractiva y proporcionar una breve introducción a la aplicación.

Contenido:

  • Bienvenida a la aplicación.
  • Promoción de los artistas.
  • Botón o enlace claro para acceder a la página de inicio de sesión.

2. Página de Contacto:

Descripción: Se debe proporcionar una forma sencilla para que los usuarios se pongan en contacto con el equipo de soporte.

Contenido:

  • Formulario de contacto con campos para nombre, correo electrónico y mensaje.
  • Botón claro para enviar el mensaje a soporte por whatapp.

3. Inicio de Sesión:

Descripción: Los usuarios deben poder acceder a la aplicación utilizando credenciales válidas.

Contenido:

  • Campos de entrada para nombre de usuario (o correo electrónico) y contraseña.
  • Enlace para restablecer la contraseña si es necesario.

4. Tabla de Incidencias:

Descripción: Interfaz para gestionar y visualizar todas las incidencias registradas.

Contenido:

  • Tabla que muestra las incidencias con columnas para ID, descripción, estado, fecha, etc.
  • Botones para agregar nueva incidencia.
  • Opciones de filtrado y clasificación para facilitar la búsqueda.

5. Crear Nueva Incidencia:

Descripción: Permitir a los usuarios registrar nuevas incidencias de manera eficiente.

Contenido:

  • Formulario con campos para descripción, prioridad, fecha, etc.
  • Validación de datos para garantizar la integridad de la información.

6. Editar Incidencia:

Descripción: Posibilidad de realizar modificaciones en una incidencia existente.

Contenido:

  • Formulario prellenado con la información existente para facilitar las modificaciones.

7. Eliminar Incidencia:

Descripción: Capacidad para eliminar incidencias que ya no son relevantes.

Contenido:

  • Botón de eliminación junto a cada incidencia en la tabla.
  • Confirmación de eliminación para evitar acciones accidentales.

Diagnóstico del Proyecto:

I. Alcance del Proyecto:

Fortalezas:

  • Alcance claro centrado en la gestión eficiente de incidencias en conciertos.

    • Oportunidades de Mejora:

    • Revisar el alcance periódicamente para adaptarse a cambios potenciales en las expectativas del usuario.
    • Explorar características adicionales que puedan mejorar la experiencia del usuario sin complicar la aplicación.

    II. Interfaz de Usuario:

    Fortalezas:

  • Planificación de una interfaz intuitiva y fácil de usar.
  • Inclusión de una tabla para visualizar y gestionar incidencias de manera eficiente.

    • Oportunidades de Mejora:

    • Explorar mejoras visuales para una experiencia de usuario más atractiva.

    III. Seguridad y Privacidad:

    Fortalezas:

  • Reconocimiento de la importancia de la seguridad y la implementación de funciones de autenticación.

    • Oportunidades de Mejora:

    • Revisión exhaustiva de las medidas de seguridad implementadas.
    • Considerar la implementación de encriptación para datos críticos.

    Conclución

      Para finalizar se puede observar que varios de los objetivos del proyecto fueron alcanzados con éxito y el cliente quedo satisfecho, para resumir estas fueron algunos de los logros luego de finalizar el proyecto:

    • Se logro el planteamiento del sistema de captura y seguimiento en tiempo real de incidentes el cual permite al festival una mejor gestión de la seguridad de los usuarios del festival.
    • Mediante el desarrollo del software se puede asegurar que todos los encargados de atender los incidentes de la aplicación puedan recopilar información de los usuarios y detalles de las incidencias que permitan una colaboración de manera inmediata.
